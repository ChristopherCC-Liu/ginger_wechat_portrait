# 微信聊天记录分析工具 · 安装指南

> ⚠️ **使用前须知**：本工具通过读取你自己设备上的微信本地数据库来分析你自己的聊天记录，
> 全程在本地运行，AI 人格分析由 Claude Code 本身完成，不需要任何 API Key，不向外部发送消息内容。
> **请勿在他人设备上操作，请勿用于分析他人数据。**

---

## 哪些步骤是自动完成的

运行 `/analyze-wechat` 命令后，以下步骤**全部自动处理**，无需手动操作：

- 安装 Python 依赖库
- 检查项目内置微信数据库解密工具和当前客户端兼容状态
- 解密微信数据库（已有密钥文件时）
- 自动查找解密后的数据库路径并写入配置
- 导出指定联系人的消息及昵称 / 头像信息
- 生成可视化图表
- 由 Claude Code 完成 AI 人格分析（双人对比）
- 生成完整 HTML 报告并在浏览器打开

## 需要你手动完成的部分（共 2 步，均为一次性）

---

### 第 1 步：关闭 SIP

SIP（System Integrity Protection，系统完整性保护）是 macOS 的安全机制，会阻止调试器附加到微信进程提取密钥。**只需关闭一次**，密钥提取完成后即可重新开启。

**操作步骤：**

1. 完全关机

2. 进入恢复模式：
   - **Apple Silicon（M 系列芯片）**：按住电源键不放，直到出现"正在加载启动选项"画面 → 点击**选项**
   - **Intel 芯片**：开机时按住 `Command + R`

3. 进入恢复模式后：顶部菜单栏 → **实用工具** → **终端**

4. 执行以下命令并重启：
   ```
   csrutil disable
   reboot
   ```

5. 重启后在普通终端中验证：
   ```bash
   csrutil status
   # 应显示：System Integrity Protection status: disabled
   ```

---

### 第 2 步：手动运行密钥提取脚本

密钥提取需要 `lldb` 调试器附加到微信进程，Claude Code 的子进程不具备此权限，**必须在系统 Terminal.app 中手动运行**（不能在 VS Code 终端、iTerm 等其他终端中运行）。

**运行 `/analyze-wechat` 后，Skill 会在需要时给出完整命令。** 你只需：

1. 打开系统自带的 **Terminal.app**（在启动台 → 其他 → 终端，或 Spotlight 搜索"终端"）

2. 在项目目录运行只读诊断，确认实际微信版本、数据库目录和 PID：

   ```bash
   cd <ginger_wechat_portrait 项目目录>
   python3 tools/wechat_db/doctor_macos.py
   ```

3. 复制 Skill 提供的项目内置命令，粘贴到 Terminal.app 执行：
   ```bash
   cd <ginger_wechat_portrait 项目目录>
   PYTHONPATH="$(lldb -P)" \
   /Library/Developer/CommandLineTools/usr/bin/python3 \
   tools/wechat_db/find_keys_macos.py --output wechat_keys.json
   ```

4. 脚本启动后会附加到微信进程，**微信界面会短暂卡住**（扫描内存期间属于正常现象）

5. 若仍有少量数据库未找到密钥，恢复微信后打开相关聊天、收藏或朋友圈页面，再次运行同一命令；已有密钥会被复用

6. 终端出现 `[+] Verified key for:` 表示密钥已通过对应数据库第一页 HMAC 校验

7. 扫描结束后，密钥自动保存为 `wechat_keys.json`

8. 执行内置批量解密：

   ```bash
   python3 tools/wechat_db/decrypt_macos.py \
     --keys wechat_keys.json --output decrypted
   ```

> **常见问题：**
> - 报 `ModuleNotFoundError: No module named 'lldb'`：必须保留 `PYTHONPATH="$(lldb -P)"` 并使用 Xcode Command Line Tools 的 Python
> - 报架构错误（Apple Silicon）：在命令前加 `arch -arm64`
> - 报 `Permission denied`：SIP 未关闭，返回第 1 步
> - 报多个 `WeChat` 进程：先加 `--list-processes` 查看，再按诊断结果加 `--pid <PID>`

---

## 密钥提取完成后：立即重新开启 SIP（推荐）

密钥文件（`wechat_keys.json`）保存后，解密操作就不再需要 SIP，**建议立刻恢复系统安全性**：

1. 重启进入恢复模式（同第 1 步 1～3）
2. 终端执行：
   ```
   csrutil enable
   reboot
   ```

**重新开启 SIP 不影响已有的密钥文件和解密后的数据库，之后所有分析完全不需要 SIP。**

---

## 第二次及之后使用

只要项目的 `decrypted/` 目录下存在已校验的 `.db` 文件，Skill 会**直接跳过密钥提取**，整个流程完全自动，你只需输入联系人名字。

如果解密数据库被清理，但密钥文件（`wechat_keys.json`）还在，Skill 会自动重新解密，无需 SIP 也无需手动操作。

## 常见问题

**Q：脚本运行时微信卡住很久，正常吗？**
A：正常。lldb 附加进程会扫描内存，耗时 2～5 分钟，等待即可。

**Q：解密后找不到数据库文件**
A：在微信里随便打开几个对话，等几秒让消息写入磁盘，再重新运行 Skill。

**Q：扫描后仍有数据库显示缺少密钥**
A：打开对应的聊天、收藏或朋友圈页面，再运行同一命令。工具会保留已有且校验通过的密钥，只补扫缺少的 salt。

**Q：Apple Silicon 报架构错误**
A：在命令前加 `arch -arm64`，或重新打开一个原生 ARM 终端窗口。

**Q：终端显示存在两个 WeChat PID**
A：运行 `python3 tools/wechat_db/doctor_macos.py`。工具会显示每个 PID 打开的数据库数量，并给出推荐 PID；扫描命令追加 `--pid <推荐值>`。

**Q：微信升级后还能继续用旧密钥吗？**
A：先运行 `doctor_macos.py` 记录新版本和数据库 salt。已存在且 HMAC 校验通过的密钥会继续复用；新增或 salt 变化的数据库才会重新扫描。微信大版本升级后应重新运行诊断和密钥扫描。

**Q：不想分析对方，只想看自己的数据**
A：正常运行即可，如果没有对方的分析结果，报告会以单人模式展示。
